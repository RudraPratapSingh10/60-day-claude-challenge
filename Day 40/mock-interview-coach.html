The system prompt

Every Claude call in this app sends the same system prompt, reproduced in full at the bottom of this panel. It does three jobs at once: it defines a narrow role (entry-level interview coach, one question at a time), it locks the output format for two distinct modes (plain-text interview turns vs. strict JSON for the report), and it pre-decides how to handle bad input — empty answers, off-topic replies, or attempts to get the model to break character — before those situations happen, rather than patching them turn by turn.

Why the interface looks like a legal pad

Interview prep is something people scribble notes for, so the UI borrows that object directly: ruled lines, a red margin rule, spiral binding, and a typewriter-style question card. It's a deliberate alternative to a generic chat bubble UI — the "tabs" across the top double as a progress tracker without needing a separate progress bar, and the final report is styled as a stamped grade on the page rather than a dashboard, which fits the "coach hands back your practice sheet" moment better than a chart-heavy analytics screen.

How the two response modes work

The app keeps the full conversation client-side and replays it on every request, since the API itself is stateless. For ordinary turns, Claude returns plain text (one acknowledgment + one question). When the session ends, the app appends a single trigger message, GENERATE_REPORT, instructing Claude to stop interviewing and instead return only JSON matching a fixed schema, which the app parses directly into the score bars, strengths, and next-step panel.

Extending this assistant

  • Memory: persist each session's report to local storage or a backend so a user can track improvement across multiple practice sessions over time.
  • Tools: give Claude a web-search tool to pull real, current interview questions for a specific company, or a resume-parsing step to tailor questions to a user's actual background.
  • Multi-step flows: add a branch where low-effort answers trigger a guided STAR-method follow-up prompt before moving to the next question, instead of just re-asking once.
  • Voice: swap the textarea for speech-to-text input to better simulate a live interview, and add a timer per answer.
